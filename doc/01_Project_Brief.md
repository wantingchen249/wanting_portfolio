## 1. Purpose

This document defines the strategic intent, scope, constraints, and success criteria for Wanting Chen's personal professional portfolio — a living digital artifact that showcases her expertise in HRIS, Data Analytics, Business Intelligence, and Digital Transformation.

The portfolio serves three audiences:

| Audience | Goal |
|---|---|
| **Recruiters / Hiring Managers** | Rapidly assess fit for senior HRIS / People Analytics roles |
| **Professional Peers** | Learn from project write-ups, frameworks, and published insights |
| **Executive Stakeholders** | Validate credibility before engaging in consulting or leadership discussions |

---

## 2. Project Goals

### Primary Goals
1. **Credibility** — Present Wanting's depth of experience through structured, evidence-based project write-ups with quantified impact.
2. **Discoverability** — Optimise content structure and metadata so the portfolio surfaces in relevant searches (HR Tech, People Analytics, Power BI).
3. **Scalability** — Build on a framework that supports continuous updates — new projects, blog posts, certifications — without rework.
4. **AI-Augmented Interaction** — Embed an AI portfolio agent that lets visitors ask questions and receive rich, sourced answers about Wanting's background.

### Secondary Goals
- Demonstrate technical fluency through the portfolio's own design and tooling choices.
- Establish a personal brand voice: *precise, data-driven, human-centred*.
- Provide shareable deep-links to individual projects for use in job applications and LinkedIn.

---

## 3. Scope

### In Scope
- Static HTML/CSS/JS portfolio site (single-page + anchored sections).
- Markdown knowledge-base documents (`knowledge/`, `projects/`, `portfolio-docs/`).
- AI persona and agent instruction set.
- Design system and component library.
- Prompt library for ongoing content generation.
- 5 flagship project case studies.

### Out of Scope (Phase 1)
- Backend CMS or database.
- Payment or booking system.
- Multi-language localisation (future: Mandarin version).
- Mobile native app.

## 5. Constraints & Principles

### Technical Constraints
- **Zero dependencies** for core site — no frameworks, no CDN reliance.
- **Accessibility** — WCAG 2.1 AA compliant (contrast ratios, keyboard nav, aria labels).
- **Performance** — Target Lighthouse score ≥ 90 across all categories.
- **Portability** — All files must render correctly when opened from local filesystem.

### Design Principles
1. **Data First** — Every claim is backed by a metric or named outcome.
2. **Clarity Over Cleverness** — Professional tone; no jargon-for-jargon's sake.
3. **Consistency** — Design tokens and content templates applied uniformly.
4. **Progressive Disclosure** — Hero gives overview; project pages give depth.

### Content Principles
- Lead with impact, follow with method.
- Use active voice; avoid passive constructions.
- Every project must include a quantified outcome.
- Chinese name (陳琬婷) appears wherever the full name is presented.
